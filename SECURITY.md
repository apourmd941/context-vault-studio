# Security Policy

## Scope

Security issues include, but are not limited to:

- access-boundary bypasses
- unintended file disclosure
- unsafe path traversal
- preview or media serving vulnerabilities
- startup-script behavior that can kill unrelated processes

## Reporting

Please report suspected vulnerabilities privately to the maintainers rather than opening a public issue first.

Until a dedicated disclosure address is published, include:

- a clear description of the issue
- affected version or commit
- reproduction steps
- expected versus actual behavior
- suggested mitigations if known

## Handling

We will aim to:

- acknowledge the report promptly
- reproduce and assess impact
- ship a fix or mitigation
- document any important operator action required
